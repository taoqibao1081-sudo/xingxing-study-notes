## 关卡1：资产负债表按科目分组汇总

### 核心逻辑
```
4家摞在一起（159行） → 过滤合计行 → 按科目分组求和 → 合并汇总表
```

### 为什么必须过滤合计行？
合计行（如"流动资产合计""负债合计"）是各公司自己算的汇总数，如果也参与 Group By，会被**重复计算**（实测差额达 5 亿）。

### PQ 操作步骤
1. **引用**「合并资产负债表」→ 改名为「资产方汇总」
2. **删除列**：去掉负债方3列，只保留 name+资产+期末+年初
3. **改列名**：资产→科目, 期末余额→期末金额, 年初余额→年初金额
4. **过滤**：取消勾选含「合计」「总计」「---」的行
5. **分组依据**：按「科目」分组，对金额列求和
6. 同样流程做「负债方汇总」：引用→删资产列→改列名→过滤→分组

### 成果验证
| 项目 | 期末 | 年初 |
|------|------|------|
| 资产合计 | 5.27 亿 | 4.70 亿 |
| 负债合计 | 5.25 亿 | 4.69 亿 |
| **差额** | **156 万**（0.3%）| **103 万** |

差额与分家原始数据一致 → 汇总逻辑正确，差额来自 302（小企业准则）科目映射残留。

## 关卡2：利润表 4 家合并

### 挑战
4 家利润表结构不统一：
- 101/207：3列，表头在第4行
- 302：4列(B列是行次编号不是金额)，表头在第3行
- 503：3列，表头在第3行

### 解决策略
**统一只保留「科目 + 本年累计」两列**，删除「本期金额」列。4家都是 2026年1-6月累计，金额可比可加。

### 操作流程
1. **建 4 家查询**：每家从 Excel 文件接入 → 删除表头行 → 提升标题 → 改列名 → 修整+过滤 → 添加工司标识列
2. **追加合并**：4家追加 → 「合并利润表」（52行）
3. **修复 BUG**：503 用英文冒号 `减:`，101/207/302 用中文冒号 `减：`，导致 Group By 拆成两行
   - 解决：在合并利润表加「替换值」步骤（`减:` → `减：`）
4. **分组汇总**：引用 → 过滤掉营业利润/利润总额/净利润/合计行 → Group By 科目求和

### 成果
| 科目 | 金额 |
|------|------|
| 营业收入 | 1.29 亿 |
| 营业成本 | 1.14 亿 |
| 管理费用 | 1,947 万 |
| 销售费用 | 370 万 |
| **合并净利润** | **-950 万** |

## 关键技术点

- **Group By = 合并底稿的「加总列」**
- **过滤合计行是分组汇总的灵魂** —— 不过滤就重复计算
- **引用查询**复用已有结果，不重复造轮子
- **替换值**解决符号不统一（英文冒号 vs 中文冒号）
- **追加前统一列结构** —— 所有表列名必须一致
