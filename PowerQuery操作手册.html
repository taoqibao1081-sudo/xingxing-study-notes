第3章 M语言三件套

第1章 五大基础操作

这5个操作是所有 PQ 工作的基础，90% 的数据清洗都用它们完成。

1.1 删除行/列

删列：选中列 → 右键 → 删除列（或主页→删除列）

删多列：Ctrl+点击多列 → 右键 → 删除列

删顶部空行：主页 → 删除行 → 删除最前面几行 → 输入行数

删底部空行：主页 → 删除行 → 删除最后几行

1.2 更改数据类型

点击列名左侧的类型图标

选择目标类型：整数/小数/文本/日期

财务类比

类型就像会计的"科目类别"——金额列必须是数字类型才能求和，日期列必须是日期类型才能排序。类型错了，后面全部出错。

1.3 拆分列

选中列 → 转换 → 拆分列 → 按分隔符

输入分隔符（如逗号、空格、反斜杠）

选择拆分位置：最左侧 / 最右侧 / 每次出现

典型场景

"应收账款-海澜餐饮" 拆成 "应收账款" 和 "海澜餐饮" 两列，方便后续按公司筛选抵消。

1.4 替换值

选中列 → 转换 → 替换值（或右键）

查找值：要替换的内容（如 null）

替换为：新内容（如 0）

常用场景

透视后出现 null 导致合计报错 → 选中所有公司列 → 替换 null 为 0。

1.5 透视列（长转宽）

选中要作为列头的列（如"公司名称"）

转换 → 透视列

值列：选要填入数据的列（如"金额"）

聚合值函数：不要聚合（每个项目只有一个值时）

注意

透视列后，如果某个公司在某些项目上没有值，会出现 null，需要替换为 0。

第2章 追加 vs 合并查询

这两个操作名字像，但功能完全不同，必须分清。

操作类比什么时候用效果
追加查询两张表上下摞起来多家公司报表格式相同，要堆在一起行数增加，列数不变
合并查询两张表左右拼起来要用另一张表的数据补信息（如科目映射）列数增加，行数不变

2.1 追加查询操作

主页 → 追加查询（或追加为新查询）

选择要追加的表（可多选）

确定 → 两张表上下拼接

2.2 合并查询操作

主页 → 合并查询

选择要合并的表

指定匹配列（两边各选一列，如"科目名称"）

联接种类：左外部（最常用，保留左边所有行）

确定 → 展开新列（点展开按钮，选需要的字段）

财务类比

追加 = 3家子公司资产负债表上下堆起来变一张大表。
合并 = 左边是原始科目，右边接上目标科目名（科目映射字典）。

第3章 M 语言三件套

M 语言的三个核心语法，掌握它们就能写出自定义逻辑。

3.1 let ... in 结构

所有 M 代码的骨架，把复杂计算拆成多个步骤。

let
    源 = Excel.Workbook(File.Contents("C:\路径\文件.xlsx"), null, true),
    表1 = 源{[Item="Sheet1",Kind="Sheet"]}[Data],
    结果 = 表1
in
    结果
财务类比

let ... in 就像做一张工作底稿——先算A步骤，再算B步骤，最后 in 出你要的那一步结果。每步都可以被后面的步骤引用。

3.2 if ... then ... else 条件判断

根据条件返回不同值，类似 Excel 的 IF 函数。

if [公司名称] = "海澜餐饮" then "母公司"
else "子公司"
if [公司名称] = "海澜餐饮" then "母公司"
else if [公司名称] = "绿茶茶饮" then "子公司A"
else "子公司B"

3.3 try ... otherwise 容错处理

防止报错中断流程，出错时返回默认值。

try Number.From([金额]) otherwise 0
财务类比

就像 Excel 的 IFERROR——试一下能不能转成数字，转不了就当 0 处理，不会因为某个单元格有文字就整列报错。

第4章 常用 M 函数速查

函数用途示例
Excel.Workbook读取 Excel 文件Excel.Workbook(File.Contents(路径), null, true)
Folder.Files读取文件夹内所有文件Folder.Files("C:\文件夹")
File.Contents读取单个文件File.Contents("C:\文件.xlsx")
Text.Contains判断文字是否包含某词Text.Contains([科目], "内部")
Text.Start取左边N个字符Text.Start([科目], 4)
Text.End取右边N个字符Text.End([科目], 4)
Text.Split按分隔符拆分文本Text.Split([科目], "-")
Number.From转成数字Number.From([金额])
Text.From转成文本Text.From([金额])
List.Sum列表求和List.Sum({1, 2, 3})
List.Skip跳过列表前N项List.Skip(列表, 1)
List.Transform对列表逐项处理List.Transform(列表, each _ * 2)
Record.FieldValues取整行所有字段值Record.FieldValues(_)
Table.SelectRows筛选行Table.SelectRows(表, each [金额] > 0)
Table.AddColumn添加列Table.AddColumn(表, "新列", each ...)
符号速记

_ = 当前行/当前项  |  [字段名] = 取当前行的某列  |  each = "对每个"  |  ?? = 如果左边是null就用右边  |  & = 字符串拼接

第5章 文件夹批量导入

把一个文件夹里的多个 Excel 文件一次性导入并合并。

操作步骤

数据 → 获取数据 → 自文件 → 自文件夹

输入文件夹路径 → 确定

弹出文件列表 → 点击 转换数据（不要点合并！）

进入 PQ 编辑器 → 保留"Content"和"Name"两列，删除其他

添加列 → 自定义列 → 公式：Excel.Workbook([Content], null, true)

展开新列 → 勾选 Data、Name → 确定

再展开 Data 列 → 勾选所有数据列 → 取消"使用原始列名作为前缀"

筛选掉表头行（取消勾选"项目"等表头文字）

重命名列、设置类型 → 完成

常见坑

第3步一定要点 转换数据，不要点"合并"。点了合并会弹一个文件选择器，只能选一个文件，失去批量意义。

第6章 参数表驱动合并

用一张 Excel 表控制"合并哪些公司"，加减公司只改表不改代码。

6.1 建 Excel 参数表

新建工作表，命名"参数表"

填入三列：公司名称 / 文件名 / 纳入合并（Y或N）

选中数据区域 → Ctrl+T → 勾选"表包含标题" → 确定

6.2 导入参数表到 PQ

选中参数表任意单元格 → 数据 → 从表格/范围

进入 PQ → 重命名查询为"公司清单"

筛选"纳入合并"列 → 只保留 Y → 确认新增"筛选的行"步骤

6.3 根据参数表批量读取文件

添加自定义列"完整路径"：文件夹路径 & [文件名]

添加自定义列"文件内容"：Excel.Workbook(File.Contents([完整路径]), null, true)

展开文件内容 → 勾选 Data → 确定

再展开 Data → 勾选所有数据列 → 确定

筛选掉表头行（取消勾选"项目"等）

删除不需要的列（文件名、完整路径等）→ 保留公司名称+项目+金额

6.4 转宽表（合并底稿格式）

选中"公司名称"列 → 转换 → 透视列

值列选"金额" → 聚合选"不要聚合" → 确定

替换 null 为 0（选中所有公司列 → 替换值）

添加动态合计列（见第12章）

使用方法

加公司：参数表添一行 + 把文件放到文件夹 + 刷新
删公司：参数表把 Y 改 N + 刷新
换数据：替换文件夹里的文件 + 刷新

第7章 双口径科目映射

把企业会计准则的科目名转换成小企业会计准则的科目名。

7.1 建科目映射字典

新建 Excel 表，两列：原科目（企业准则）/ 目标科目（小企业准则）

如：应收账款 → 应收账款；其他应收款 → 其他应收款；预付款项 → 预付账款

Ctrl+T 转为表格 → 导入 PQ

7.2 合并查询完成映射

在原始数据查询中 → 主页 → 合并查询

选择"科目映射"表 → 左右各选"科目名称"列匹配

联接种类：左外部（保留所有原始行）

确定 → 展开新列 → 只勾选"目标科目"

检查映射结果：原科目列旁边出现目标科目列

7.3 验证映射完整性

必须检查

展开后如果"目标科目"列出现 null，说明映射字典里漏了这个科目。回 Excel 补上，刷新即可。

第8章 合并抵消 · 场景1：内部往来

母子公司之间的应收应付相互抵消。

业务逻辑

母公司子公司抵消方向
其他应收款-子公司（资产↑）其他应付款-母公司（负债↑）双方对冲归零

操作步骤

把所有公司数据追加为一张长表（公司名称+科目+金额）

添加条件列"是否内部往来"：if Text.Contains([科目], "内部") then "Y" else "N"

筛选"是否内部往来"= Y 的行

添加抵消列：母公司行金额取负数，子公司行金额保持正数

把抵消行追加回原始数据 → 合并报表中内部往来自动归零

if Text.Contains([科目], "内部") then
    if [公司名称] = "海澜餐饮" then -[金额]
    else [金额]
else 0
核心原理

内部往来 = 一方的应收 = 另一方的应付。金额相等方向相反，加起来 = 0。母公司取负、子公司取正，追加回去就自动抵消。

第9章 合并抵消 · 场景2：内部交易

母公司向子公司销售（或采购），合并层面要抵消内部收入和对应的内部成本。

业务逻辑

母公司子公司抵消效果
主营业务收入-内部销售主营业务成本-内部采购收入↔成本对冲

操作步骤

在合并数据中，用 Text.Contains 自动识别含"内部"的科目

内部收入行：金额取负数（冲减收入）

内部成本行：金额取负数（冲减成本）

把抵消行追加回原始数据 → 合并层面收入和成本同时减少

if Text.Contains([科目], "内部销售收入") then -[金额]
else if Text.Contains([科目], "内部采购成本") then -[金额]
else 0
核心原理

集团内部的销售，从合并视角看就是"左手卖右手"，不算真正的收入。收入减、成本也减，利润不变，但收入成本规模更真实。

第10章 合并抵消 · 场景3：股权投资

母公司长期股权投资与子公司所有者权益相互抵消，确认少数股东权益。

业务逻辑

抵消项母公司子公司方向
长期股权投资 ↔ 实收资本+资本公积+未分配利润长投（资产）所有者权益双方对冲
少数股东权益—权益 × 少数股东比例差额确认

操作步骤

添加条件列，科目+公司双重判断

母公司"长期股权投资"行：金额取负数

子公司"实收资本/资本公积/未分配利润"行：按持股比例取负数

计算少数股东权益 = 子公司净资产 × (1 - 持股比例)

添加少数股东权益行（正数，作为合并权益的组成部分）

所有抵消行追加回原始数据

if [科目] = "长期股权投资" and [公司名称] = "海澜餐饮" then -[金额]
else if Text.Contains([科目], "实收资本") and [公司名称] <> "海澜餐饮" then -[金额] * 0.8
else if Text.Contains([科目], "资本公积") and [公司名称] <> "海澜餐饮" then -[金额] * 0.8
else if Text.Contains([科目], "未分配利润") and [公司名称] <> "海澜餐饮" then -[金额] * 0.8
else 0
关键参数

上面的 0.8 是母公司持股比例（80%），请根据实际投资比例修改。1 - 0.8 = 0.2 就是少数股东权益比例。

核心原理

母公司长投 = 子公司净资产 × 持股比例。合并时长投和对应权益对冲掉，剩余的权益部分 = 少数股东权益，作为合并报表的权益项目保留。

第11章 路径参数化

把写死的文件路径变成可改的参数，换文件夹不用改代码。

操作步骤

PQ 编辑器 → 管理参数 → 新建参数

名称：文件夹路径，类型：文本，当前值：填实际路径

确定 → 左侧出现参数

回到数据查询 → 点"源"步骤 → 修改公式栏

= Excel.Workbook(File.Contents("C:\固定路径\文件.xlsx"), null, true)
= Excel.Workbook(File.Contents(文件夹路径 & "\文件名.xlsx"), null, true)
常见坑：路径拼接少反斜杠

如果参数值结尾没有 \，拼接时必须在文件名前加 \。否则 场景2海澜.xlsx 会变成一个无效路径。
建议：参数值以 \ 结尾，公式里就不用再加了。

第12章 动态合计公式

透视列后，公司数量会变化，固定列名合计公式会报错。用动态公式一劳永逸。

问题根源

写死列名 = 僵化

[海澜餐饮] + [绿茶茶饮] → 当绿茶被取消合并时，这列不存在了，公式报错。

解决方案

不写列名，动态取整行所有数字列求和。相当于 M 语言版的 =SUM(B2:Z2)。

List.Sum(List.Transform(List.Skip(Record.FieldValues(_), 1), each _ ?? 0))

公式拆解

部分作用Excel 类比
Record.FieldValues(_)取当前行所有字段值，变成列表把整行数据抓出来
List.Skip(..., 1)跳过第1个（"项目"是文字，不能加）跳过A列的科目名
each _ ?? 0把 null 转成 0空格当0处理
List.Sum(...)全部求和=SUM(...)

使用方法

添加列 → 自定义列

新列名：合计

粘贴上面的公式 → 确定

验证：改参数表 Y/N → 刷新 → 合计自动重算，不报错

为什么这个公式是"自动化"的核心

它一个列名都不写！不管未来3家、5家、30家公司，它都自动取当前行所有数字列求和。这是从"写死"到"动态"的关键一跃。

第14章 常见报错排查

14.1 路径拼接报错

症状：DataFormat.Error · 找不到文件

原因：路径拼接少反斜杠，变成 场景2海澜.xlsx 而不是 场景2\海澜.xlsx
修复：参数值结尾加 \，或公式里 & "\文件名"

14.2 透视列后合计报错

症状：找不到列名 / Expression.Error

原因：合计公式写死了列名 [公司A]+[公司B]，公司被取消后列消失
修复：改用第12章的动态合计公式

14.3 透视列出现 null

症状：合计列全是 null 或部分行是 null

原因：某公司在某些项目上没数据，透视后变成 null，null + 数字 = null
修复：选中所有公司列 → 替换值 null → 0

14.4 参数表改了但不生效

症状：参数表改了 Y/N，刷新后数据没变

原因：PQ 里"公司清单"查询没有添加"筛选的行"步骤
修复：进 PQ → 选中"公司清单" → 筛选"纳入合并"列只保留 Y

14.5 导入后多出表头行

症状：数据最后几行出现"项目""金额"文字行

原因：每个 Excel 文件的第一行是表头，导入时混进来了
修复：筛选"项目"列 → 取消勾选"项目"等表头文字 → 确定

14.6 合并查询后目标列出现 null

症状：科目映射后，部分行目标科目是 null

原因：映射字典里漏了这些科目
修复：回 Excel 参数表补上缺失的映射 → 刷新

14.7 刷新时报"键不能为 null"

症状：刷新时弹出键值错误

原因：合并查询的匹配列里有 null 值
修复：在合并前先筛选掉匹配列为 null 的行，或替换为默认值
