## 三、模型2：Python 一键对账

### 核心逻辑

```python
df_a = pd.read_excel("总账.xlsx")     # 打开账本A
df_b = pd.read_excel("明细账.xlsx")   # 打开账本B

# outer join → 所有科目都显示，两边都有就并排
df = pd.merge(df_a, df_b, on="科目", how="outer")

# 差异 = 明细账金额 - 总账金额
df["差异"] = df["金额_B"] - df["金额_A"]

# 分类标注：MATCH / AMOUNT_DIFF / ONLY_A / ONLY_B
# 输出彩色 Excel（绿色一致/红色金额差异/黄色缺失）
```

### 用到的 pandas 函数（你都学过）
- `pd.read_excel()` — 读文件
- `pd.merge(how="outer")` — 按科目对齐
- `fillna(0)` — 空值填0
- 自定义分类函数 — 打标签
- `openpyxl` 写带颜色的 Excel

### 测试验证
用 101 资产负债表造了 4 个差异（金额不一致×2 + 缺失×1 + 多余×1），脚本全部命中。

## 四、模型3：财务体检仪表盘

### 指标计算

从合并数据中按科目关键词分类计算：

| 类别 | 关键词 |
|---|---|
| 流动资产 | 货币资金/应收/预付/存货/其他应收… |
| 非流动资产 | 固定资产/无形资产/长期股权/在建工程… |
| 流动负债 | 短期借款/应付/预收/应付职工薪酬… |
| 所有者权益 | 实收资本/资本公积/盈余公积/未分配利润 |

### 8 项财务指标

| 指标 | 公式 | 合并结果 | 健康线 | 状态 |
|---|---|---|---|---|
| 流动比率 | 流动资产 ÷ 流动负债 | 0.97 | >1.5 | !! |
| 速动比率 | (流动资产-存货) ÷ 流动负债 | 0.92 | >1.0 | !! |
| 资产负债率 | 总负债 ÷ 总资产 | 99.7% | <60% | !! |
| 毛利率 | (营收-成本) ÷ 营收 | 11.3% | >20% | !! |
| 净利率 | 净利润 ÷ 营收 | -7.4% | >5% | !! |
| 管理费用率 | 管理费用 ÷ 营收 | 15.1% | <10% | !! |
| 应收周转率 | 营收 ÷ 应收账款 | 4.25次 | >3次 | OK |
| 现金覆盖比 | 货币资金 ÷ 短期借款 | 0.21 | >0.5倍 | !! |

### 可视化内容
- 4 张总览卡片（总资产/总负债/营收/净利润）

## 五、踩坑记录

### 坑1：Python dict 同名键覆盖
- 问题：资产方汇总里有同名科目（空格不同的重复），直接用 dict 会覆盖
- 解决：改用 `defaultdict(float)` + `+=` 累加

### 坑2：利润表成本是正数
- 问题：以为成本是负数，直接加导致 gp = revenue + cost 膨胀
- 解决：成本/费用都取 `abs()`，计算时用减法

### 坑3：f-string 与 JavaScript 花括号冲突
- 问题：Python f-string 的 `{变量}` 和 JS 的 `{…}` 语法冲突
- 解决：用 JSON 注入数据，HTML 模板用 `replace("__JSON_HERE__", json_str)`
