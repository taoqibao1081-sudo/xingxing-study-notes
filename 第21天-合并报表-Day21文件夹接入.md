## 三、操作步骤

### 1. 建合并工作台
- `第二阶段` 目录 → 新建 Excel → 命名 `合并工作台.xlsx`

### 2. 从文件夹接入
- 「数据」→「获取数据」→「从文件」→「从文件夹」→ 选择 `报表` 子文件夹

### 3. 进 PQ 编辑器 + 瘦身
- 点「转换数据」进入 Power Query 编辑器
- Ctrl 选中 3 列（Date accessed / Date created / Attributes）→ 右键删除
- 保留 5 列：Content / Name / Extension / Date modified / Folder Path

### 4. 改名 + 加载
- 查询改名：「报表」→「报表清单」
- 「主页」→「关闭并应用」加载到 Excel

### 5. 验证
- 往 `报表` 文件夹丢一个空 xlsx → Excel 点「全部刷新」
- 「报表清单」从 8 行变 9 行 = 过关 ✅
- 删掉测试文件，刷新回到 8 行

## 四、关键理解

- **Content 列显示 Binary**：因为 xlsx 是压缩包，PQ 暂时只看到"压缩包外壳"。**Day22 要展开这一列**才能读到里面的资产负债表数据。
- **参数化 = 路径可换**：文件夹路径是活的参数，换文件夹只改路径，扫描流程不动。
- **PQ 自动记录每一步**：右侧"应用的步骤"会显示每一步的 M 代码（如 `Table.RemoveColumns`），相当于审计痕迹。
