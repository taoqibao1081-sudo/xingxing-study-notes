## ✅ 第0关：Python 函数速成

### 核心三件套

| 概念 | 财务类比 | 一句话 |
|------|----------|--------|
| `def 函数名():` | Excel 自定义公式 | 把逻辑打包起个名字 |
| 参数 `()` | 公式里的单元格引用 `=A1, A2` | 往里传数据 |
| `return` | 公式算出的结果 | **关键！**把结果交出去继续用 |

### ⚠️ print vs return 区别

```python
# print 只是在屏幕上"显示一下"，拿不到值
def 税率1(金额):
    print(金额 * 0.13)   # ❌ 拿不到这个数

# return 把结果"交出去"，别人能继续用
def 税率2(金额):
    return 金额 * 0.13   # ✅ 可以赋值给变量
```

### 综合实战：子公司现金盘点表

```python
def 现金盘点(公司名, 账面金额, 实盘金额):
    差异 = 实盘金额 - 账面金额
    if abs(差异) <= 1:
        状态 = "✅ 平账"
    elif 差异 > 1:
        状态 = "⚠️ 盘盈"
    else:
        状态 = "❌ 盘亏"
    return {"公司": 公司名, "差异": 差异, "状态": 状态}
```

## ✅ 关卡1：System Prompt 角色设定

### 核心一句话
**System Prompt = 给 AI 发的"岗位说明书"**——写得越细，AI 越懂你想要什么。

### 同题异答演示：3 个角色回答"内部交易抵消"

| 角色 | 风格 | 适合场景 |
|------|------|---------|
| 默认 AI | 简洁直接 | 快速查询 |
| **老财风格** | 三步法 + 口诀 + 类比 | 日常工作首选 |
| 学院教授 | 准则条款号 + 学术规范 | 写报告、应对审计 |

### 自己改的"老财风格"提示词（5 条规则全生效）

```python
"system": """你是老财，幸幸的专属财务搭档。

【你的风格】
- 务实第一，幽默第二。专业的事必须严谨。
- 说话接地气，像个靠谱的财务老炮儿，不端着。
- 能用财务类比解释技术问题（比如把函数比作Excel公式）。
- 不确定的事直说，不编数字，不许拍脑袋。

【回答要求】
- 先给结论，再讲原理，最后举例子。
- 用"三步法"：原理→操作→验证。
- 涉及金额必须标注"元/万元"单位。
- 用【】和✅❌⚠️等符号让结构清晰。"""
```

### 💡 小技巧：正面指令 > 负面禁令

```python
# ❌ AI 偶尔会漏掉
"不许用emoji"

# ✅ 更稳
"用【】和 ✅⚠️ 等文本符号分点，禁止使用任何 emoji"
```

## ✅ 关卡2A：认识 JSON 工具清单

### 核心一句话
**Tools = 公司给员工发的"工具箱"**。AI 看到工具清单后，**会自己判断**该用哪个工具。

### Tool 三件套 JSON 结构

```json
{
  "type": "function",
  "function": {
    "name": "calculate_depreciation",  ← 工具名（必填英文）
    "description": "何时用这个工具的说明",
    "parameters": {
      "type": "object",
      "properties": {
        "原值": {"type": "number", "description": "设备原值"},
        "使用年限": {"type": "integer", "description": "使用年限"}
      },
      "required": ["原值", "使用年限"]
    }
  }
}
```

### ⚠️ 踩坑：DeepSeek 不接受中文 tool name

报错：`Invalid 'tools[0].function.name': string does not match pattern. Expected a string that matches the pattern '^[a-zA-Z0-9_-]+$'`

**解决**：API 层的 `name` 用英文，内部 Python 函数名可以保留中文。

## ✅ 关卡2B：亲手给 Agent 装第二只手

### 我写的增值税计算器

```python
def calculate_vat(金额, 纳税人类型, 业务类型):
    税率表 = {
        ("一般纳税人", "销售货物"): 0.13,
        ("一般纳税人", "交通运输"): 0.09,
        ("一般纳税人", "现代服务"): 0.06,
        ("小规模纳税人", "销售货物"): 0.03,
        ("小规模纳税人", "现代服务"): 0.03,
    }
    税率 = 税率表.get((纳税人类型, 业务类型))
    if 税率 is None:
        return {"错误": f"未找到{纳税人类型}+{业务类型}的税率"}
    税额 = 金额 * 税率
    return {
        "金额": 金额, "纳税人类型": 纳税人类型,
        "业务类型": 业务类型, "税率": 税率, "税额": round(税额, 2)
    }
```

### 4 个测试全过 🎉

| 测试 | 问题 | AI 调的工具 | 结果 |
|------|------|------------|------|
| 1 | 一般纳税人卖 10 万货 | calculate_vat | ¥13,000（13%）|
| 2 | 小规模提供现代服务 5 万 | calculate_vat | ¥1,500（3%）|
| 3 | 设备原值 10 万用 5 年 | calculate_depreciation | ¥20,000 |
| 4 | 什么是会计分录？ | 不调 | 直接答 |

**核心收获**：AI **自己选对工具、自己填对参数、自己算对数**。

## 🐛 今日踩坑记录

| 序号 | 报错 | 原因 | 解决 |
|------|------|------|------|
| 1 | UnicodeEncodeError gbk | Windows 默认编码 | 加 `-X utf8` 参数 |
| 2 | ModuleNotFoundError requests | 3.13 没装 requests | 装到 venv：`pip install requests` |

| 5 | SyntaxError closing parenthesis | JSON 缺一个 `}` | 补一个右大括号 |
