## Part 1：DAX时间智能入门 ✅

### 核心函数：TOTALYTD
- **作用**：计算"年初至今累计"，就像利润表的"本年累计"列
- **公式**：`累计收入 = TOTALYTD(SUM('月度收入'[收入]), '日期表'[Date])`

### 关键步骤回顾

#### 1. 新建日期列（月度收入表）
**问题发现**：月份列实际格式是 `"2026年1月1日"` 中文日期文本，不是之前以为的 `"1月"` 简写。

**排坑过程**：
| 尝试 | 公式 | 结果 |
|:---|:---|:---|
| 第一次 | `DATEVALUE('月度收入'[月份])` | ❌ 不认中文格式 |
| 第二次 | `FIND("月", ...)` | ❌ 中文兼容性问题 |
| **最终方案** | `DATE(VALUE(LEFT('月度收入'[月份],4)), VALUE(MID('月度收入'[月份],6,2)), 1)` | ✅ 成功 |

**最终公式**：
```dax
日期 = DATE(
    VALUE(LEFT('月度收入'[月份], 4)),     // 取前4位：2026
    VALUE(MID('月度收入'[月份], 6, 2)),   // 取第6-7位：01、02
    1                                      // 日固定为1号
)
```

#### 2. 确认日期表已存在
日期表上午已创建，包含：Date、年份、月编号、月份、年月等字段，无需重建。

#### 3. 模型视图建立关系
- **关系**：日期表[Date] → 月度收入[日期]
- **类型**：一对多（日期表是唯一值侧）

#### 4. 创建 TOTALYTD 度量值并验证
```dax
累计收入 = TOTALYTD(SUM('月度收入'[收入]), '日期表'[Date])
```

**验证结果**：
| 年月 | 当月收入 | 累计收入 | 验证计算 |
|:---|---:|---:|:---|
| 2026年01月 | 260,000 | 260,000 | ✓ |
| 2026年02月 | 265,000 | 525,000 | 260000+265000=525000 ✓ |

## Part 2：同比 & 环比 ✅

### 时间智能三剑客

| 指标 | 英文名 | DAX函数 | 用途 |
|:---|:---|:---|:---|
| **上年收入** | Previous Year (PY) | `SAMEPERIODLASTYEAR` | 取去年同期的值 |
| **同比增长率** | Year-over-Year (YOY) | `(今年-去年)/去年` | 今年 vs 去年同月 |
| **环比增长率** | Month-over-Month (MOM) | `PREVIOUSMONTH` | 本月 vs 上月 |

### 度量值公式

#### 1. 上年收入
```dax
上年收入 = CALCULATE(
    SUM('月度收入'[收入]), 
    SAMEPERIODLASTYEAR('日期表'[Date])
)
```

#### 2. 同比增长率
```dax
同比增长率 = 
VAR CurrentYear = SUM('月度收入'[收入])
VAR LastYear = [上年收入]
RETURN
    IF(
        LastYear = 0,
        BLANK(),
        DIVIDE(CurrentYear - LastYear, LastYear)
    )
```

#### 3. 环比增长率
```dax
环比增长率 = 
VAR CurrentMonth = SUM('月度收入'[收入])
VAR LastMonth = CALCULATE(
    SUM('月度收入'[收入]), 
    PREVIOUSMONTH('日期表'[Date])
)
RETURN
    IF(
        LastMonth = 0,
        BLANK(),
        DIVIDE(CurrentMonth - LastMonth, LastMonth)
    )
```

### 排坑：DAX变量不支持中文名！

**错误示范**：
```dax
VAR 今年 = SUM(...)
VAR 去年 = [上年收入]
```
→ 报错：语法错误

**正确写法**：
```dax
VAR CurrentYear = SUM(...)
VAR LastYear = [上年收入]
```
→ ✅ 通过

**记忆口诀**：DAX变量用英文，就像科目代码不能用汉字。

| 2026年01月 | 260,000 | (空) | (空) | (空) | 260,000 |

## 今日排坑记录（5则）

1. **DATEVALUE不认中文格式**：`"2026年1月1日"` 转日期失败 → 改用 `DATE + VALUE + LEFT/MID` 手动拼

2. **DAX FIND中文兼容性差**：`FIND("月", ...)` 报错 → 改用固定位置 `MID(月份, 6, 2)` 取月数

3. **DAX变量不支持中文**：`VAR 今年` 报错 → 改为英文 `VAR CurrentYear`

4. **数据格式与预期不符**：以为月份是 `"1月"` 简写，实际是 `"2026年1月1日"` 完整格式 → 先观察数据再写公式

5. **日期表撞名**：尝试新建"日期表"时提示已存在 → 发现上午已创建，直接使用即可
