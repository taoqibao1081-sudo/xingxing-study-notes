## 四、踩坑资产（最宝贵）

| 坑 | 根因 | 解法 | 来源 |
|----|------|------|------|
| Git Bash 中文用户名引号闭合失败 | 编码问题 | 改用英文用户名 | 单元1 |
| Excel 2016 不支持 LAMBDA | 版本 INFO("RELEASE")=16.0 | 改用 WPS | 单元4 |
| RANKX 全排第 1 | 第一参数误用 ALL(表) | 改为 ALL('公司信息'[公司名称]) | 单元4 |
| 本地 master ≠ 远程 main | 分支名不一致 | git push -u origin master | 单元2 |
| pbix 直接脚本注入条件格式损坏 | Power BI 内部 JSON schema 不公开 | 改手动在格式窗格设规则 | 单元20 |
| 条件格式规则是上下界格式 | 默认"如果值介于" | >0→最大 表正数；>最小→<0 表负数 | 单元20 |
| 矩阵底部总计重复计算 | 自动总总计 | 关闭行/列小计总计 | 单元20 |
| 同比卡片为 0 | 日期字段 DAX 匹配异常 | 增加独立「年份」整数字段 | 单元20 |
| Power BI Service 连不上 | 网络/环境限制 | 用户决定永久省掉此步 | 单元20 |
